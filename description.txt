Description of the Project: 

This project demonstrates image steganography using the Least Significant Bit (LSB) technique in 
C language. It enables users to hide and retrieve secret data within BMP images without altering 
their visible quality. The system includes both encoding and decoding operations, validated through 
command-line arguments for robustness. A unique magic string (#*) ensures secure identification of 
stego images during decoding. The project effectively showcases key programming concepts such as 
file handling, bit manipulation, and data security, providing a practical understanding of how 
confidential information can be embedded within digital media for secure communication.

ENCODING MODULE OVERVIEW  encode.c file

Importance:
Handles the process of concealing a secret file within a BMP image using LSB encoding.
Maintains the original image structure while embedding secret data securely.
Provides modular steps for validation, encoding, and final output creation.

Algorithm:
Validate command-line inputs and open all required files.
Embed metadata and file contents into the image’s pixel bytes.
Copy unused bytes to the output file and close all opened streams.

Name of the function used: get_image_size_for_bmp

Importance:
Determines the total pixel data area available for hiding information.
Uses image width and height from the BMP header to compute capacity.
Ensures correct calculation of available bits for encoding.

Algorithm:
Move to byte 18 in the BMP header to read width and height.
Fetch 4 bytes each for width and height values.
Multiply width × height × 3 to obtain total image size.

Name of the function used: open_files

Importance:
Opens all files necessary for encoding operation.
Ensures read/write access for image and secret files before processing.
Displays detailed error messages when file access fails.

Algorithm:
Open source image in read-binary mode.
Open secret file in read-binary mode.
Create a new stego image in write-binary mode.

Name of the function used: read_and_validate_encode_args

Importance:
Verifies correctness of filenames and their extensions before encoding.
Prevents invalid or missing arguments that could interrupt the process.
Automatically creates a default output file when not specified.

Algorithm:
Check if image ends with .bmp and secret file has a valid format.
Validate extensions like .txt, .c, .sh, or .h.
Assign default output filename if user doesn’t provide one.

Name of the function used: check_capacity

Importance:
Confirms whether the cover image can store the entire secret file.
Prevents overflow by comparing image capacity with required bits.
Maintains encoding integrity by ensuring enough space is available.

Algorithm:
Calculate total image capacity using width and height.
Estimate number of bits needed for the secret and metadata.
Return success only if image capacity exceeds requirement.

Name of the function used: copy_bmp_header

Importance:
Preserves the BMP image structure during the encoding process.
Ensures header information like dimensions remains unchanged.
Serves as a base before data embedding begins.

Algorithm:
Move to start of the image file.
Read the first 54 bytes (BMP header).
Write these bytes unchanged into the output stego image.

Name of the function used: encode_byte_to_lsb

Importance:
Core logic for hiding secret bits inside image pixels.
Uses the least significant bit of each image byte to store data.
Guarantees minimal visual distortion in the encoded image.

Algorithm:
Extract each bit of the secret byte using bit-shifting.
Clear the LSB of the image byte and insert the data bit.
Repeat for all 8 bits to encode one full byte.

Name of the function used: encode_magic_string

Importance:
Inserts a predefined pattern to mark an image as stego.
Acts as an identifier during decoding to verify validity.
Provides the first level of validation before extraction.

Algorithm:
Read 8 bytes from the source image for each character.
Embed every bit of the magic string using the LSB method.
Write the modified bytes into the stego image.

Name of the function used: encode_secret_file_extn_size

Importance:
Stores the length of the secret file’s extension inside the image.
Helps the decoder identify how many bytes to read for extension.
Ensures accurate file type reconstruction during decoding.

Algorithm:
Read 32 bytes from the image for storing the size.
Replace each LSB with bits representing the extension length.
Write the modified bytes to the output file.

Name of the function used: encode_secret_file_extn

Importance:
Embeds the actual file extension (for example .txt or .c) into the image.
Ensures correct file type restoration during decoding.
Keeps metadata hidden yet retrievable.

Algorithm:
For every character in the extension, read 8 image bytes.
Replace each LSB with bits of the character.
Write modified bytes to the output stego image.

Name of the function used: encode_secret_file_size

Importance:
Saves the total size of the secret file for decoding reference.
Enables precise extraction by informing the decoder about data length.
Avoids reading extra or insufficient bytes during retrieval.

Algorithm:
Read 32 bytes from the source image.
Store each bit of the file size into the LSBs of those bytes.
Write the modified bytes to the stego image.

Name of the function used: encode_secret_file_data

Importance:
Contains the main logic for hiding the secret file’s content.
Encodes each byte of the secret file into 8 image bytes.
Ensures complete concealment of text or binary data.

Algorithm:
Read a byte from the secret file.
Embed its bits into 8 bytes of the image using LSB.
Repeat until the full file is encoded.

Name of the function used: copy_remaining_img_data

Importance:
Preserves the original pixels not used for encoding.
Prevents data corruption by copying leftover bytes as-is.
Maintains image completeness after encoding is finished.

Algorithm:
Read remaining bytes from the source image.
Write them directly to the stego image.
Continue until the end of file is reached.

Name of the function used: do_encoding

Importance:
Acts as the main controller for the entire encoding sequence.
Manages file handling, size calculation, and all embedding operations.
Displays informative logs for each phase of encoding.

Algorithm:
Open all files and calculate secret file size.
Encode metadata (magic string, extension, size) followed by data.
Copy leftover image data and close all files.
